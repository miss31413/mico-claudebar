01
Prerequisite · 前置需求
安裝 jq
jq 是用來讀取資料的小工具，mico-claudebar 需要它才能運作。
Install jq — a small JSON parser that mico-claudebar depends on.
bash
brew install jq
看到 command not found: brew？請先到 brew.sh 安裝 Homebrew，網站首頁有一行指令複製貼上即可。
If you see this error, install Homebrew first at brew.sh
02
Download · 下載
下載腳本
把 mico-claudebar 下載到 Claude Code 的設定資料夾。
Download the script into Claude Code's config folder.
bash
curl -o ~/.claude/statusline.sh \ https://raw.githubusercontent.com/miss31413/mico-claudebar/main/statusline.sh chmod +x ~/.claude/statusline.sh
chmod +x 是告訴電腦「這個檔案可以被執行」，少了這步會出現權限錯誤。
This gives the script executable permission — required to run it.
03
Config · 設定
修改 settings.json
告訴 Claude Code 使用我們的狀態列腳本。
Tell Claude Code to use your statusline script.
bash — open file
open ~/.claude/settings.json
json — paste this · 貼入這段內容
{ "statusLine": { "type": "command", "command": "~/.claude/statusline.sh" } }
⚠️ 如果 settings.json 裡已經有其他設定，不要整個覆蓋——只把 "statusLine" 這段加進去。
Don't overwrite existing settings — just add the "statusLine" block.
04
Verify · 驗證
測試是否正常
不用重啟，先在終端機確認腳本正確運作。
Test the script before restarting Claude Code.
bash
'{"model":{"display_name":"Claude Sonnet 4"}, "workspace":{"current_dir":"/Users/test/myproject"}, "context_window":{"used_percentage":75}}' \ | ~/.claude/statusline.sh
Expected output · 應該看到：
◆ Claude Sonnet 4▸ myproject▓ [███████░░░] 75%
05
Done · 完成
重啟 Claude Code
關掉 Claude Code 再重新開啟，狀態列就會出現在畫面最下方。
Restart Claude Code and the statusline appears at the bottom.

01
Download · 下載
下載 PowerShell 腳本
Windows 版使用 PowerShell 腳本，不需要安裝 jq。
Windows uses a PowerShell script — no jq required.
⚠️ 需要 PowerShell 7（pwsh），不是 Windows 內建的 PowerShell 5。
下載：aka.ms/powershell（免費，安裝後電腦上會多一個 pwsh 指令）
打開 PowerShell 7（開始選單搜尋「pwsh」）
Open PowerShell 7 from the Start menu — search "pwsh"
powershell
# 建立資料夾（已存在也沒關係） New-Item -ItemType Directory -Path "$env:USERPROFILE\.claude" -Force # 下載腳本 curl -o "$env:USERPROFILE\.claude\statusline.ps1" \ https://raw.githubusercontent.com/miss31413/mico-claudebar/main/statusline.ps1
02
Permission · 權限
允許執行腳本
Windows 預設會擋住腳本執行，需要執行一次這個指令解除限制。
Allow PowerShell scripts to run — required once per machine.
powershell
Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser
這只影響你自己執行的腳本，不影響系統安全性。
This only affects scripts you run yourself — not a system security risk.
03
Config · 設定
修改 settings.json
設定檔在 %USERPROFILE%\.claude\settings.json
powershell — open file
notepad "$env:USERPROFILE\.claude\settings.json"
json — paste this · 貼入這段
{ "statusLine": { "type": "command", "command": "pwsh -NoProfile -File %USERPROFILE%\.claude\statusline.ps1" } }
04
Verify · 驗證
測試腳本
powershell
'{"model":{"display_name":"Claude Sonnet 4"}, "workspace":{"current_dir":"C:\\Users\\test\\myproject"}, "context_window":{"used_percentage":75}}' ` | pwsh -NoProfile -File "$env:USERPROFILE\.claude\statusline.ps1"
Expected output · 應該看到：

Troubleshooting · 常見問題
jq: command not found
Step 1 的 jq 沒有安裝成功，回去重新執行安裝指令。
jq wasn't installed successfully — redo Step 1.
permission denied
重新執行 chmod +x ~/.claude/statusline.sh（macOS/Linux）
Re-run the chmod command from Step 2.
狀態列沒有出現 · Statusline not showing
確認 settings.json 的 JSON 格式正確，可以把內容貼到 jsonlint.com 檢查。
Validate your settings.json at jsonlint.com
Windows — 出現紅字執行錯誤
確認使用的是 pwsh（PowerShell 7），不是 powershell（PowerShell 5）。
Make sure you're using pwsh (PowerShell 7), not the built-in powershell.
